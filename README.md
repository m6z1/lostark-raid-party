# 로아 레이드팟 짜기

이름 · 캐릭터(딜러/서폿) · 아이템레벨 · 전투력을 입력하면 4막 / 종막 / 세르카 / 지평의 성당 레이드팟을 자동으로 짜주는 단일 페이지 웹앱.

## 실행

`index.html`을 브라우저에서 열면 끝. 서버 불필요, 데이터는 브라우저 localStorage에 저장된다.

```
open index.html
```

## 레이드 입장 아이템레벨 기준 (2026-07)

| 레이드 | 인원 | 난이도 |
|---|---|---|
| 4막 파멸의 성채 | 8인 | 노말 1700 / 하드 1720 |
| 종막 최후의 날 | 8인 | 노말 1710 / 하드 1730 |
| 세르카 (그림자 레이드) | 4인 | 노말 1710 / 하드 1730 / 나이트메어 1740 |
| 지평의 성당 (어비스 던전) | 4인 | 1단계 1700 / 2단계 1720 / 3단계 1750 |

기준이 바뀌면 `index.html`의 `RAIDS` 상수만 수정하면 된다.

## 전투정보실 자동 불러오기

멤버 패널의 "🔍 전투정보실에서 자동 불러오기"에서 대표 캐릭터명 하나만 검색하면 원정대 전체 캐릭터의 클래스 · 아이템레벨 · 전투력을 로스트아크 공식 오픈 API로 가져온다. 방문자는 API 키를 입력하지 않는다.

- 운영자가 [developer-lostark.game.onstove.com](https://developer-lostark.game.onstove.com)에서 발급한 키는 Cloudflare Worker의 `LOSTARK_API_KEY` Secret으로만 보관
- 키는 저장소·배포 파일·브라우저에 포함되지 않음
- 1700 이상 캐릭터가 기본 선택되고, 바드/홀리나이트/도화가/발키리는 서폿으로 자동 분류 (딜폿은 목록에서 딜러로 바꾸면 됨)

### API 프록시 배포

1. `cd worker && npm install`
2. `npx wrangler login`
3. `npm run deploy`
4. 새로 재발급한 로스트아크 API 키를 `npm run secret`으로 등록
5. 배포된 `https://...workers.dev` 주소를 루트의 `proxy-config.js`에 입력

노출된 키는 재사용하지 않는다. 로컬 개발용 키가 필요하면 `worker/.dev.vars`에 `LOSTARK_API_KEY=...` 형식으로 저장하며 이 파일은 Git에서 제외된다.

## 팟 배정 규칙

- 선택한 난이도의 입장 레벨 이상인 캐릭터만 배정
- 8인 팟은 서폿 2 + 딜러 6 (파티당 서폿 1), 4인 팟은 서폿 1 + 딜러 3
- 같은 사람의 캐릭터 두 개가 한 팟에 들어가지 않음 (다른 팟에는 가능)
- 전투력(게임에 표시되는 점수 그대로, 선택 입력)이 높은 캐릭터끼리 같은 팟에 묶이고, 8인 팟의 1/2파티는 딜러 전투력 합이 비슷하도록 분배됨
- 전투력을 안 넣은 캐릭터는 아이템레벨 기준으로 정렬됨
- 4인 파티는 서로 다른 클래스 시너지를 먼저 채우고, 8인 공격대는 두 파티의 시너지 종류를 최대화한 뒤 전투력 합 차이가 가장 작은 조합을 선택
- 각 파티 결과와 복사 텍스트에 적용 가능한 시너지 종류를 표시 (클래스를 입력하지 않은 기존 데이터도 그대로 사용 가능)
- 시너지 분류는 딜러의 공통 핵심 공격 시너지 기준이며, 서포터의 낙인과 각인·트라이포드에 따른 부가 시너지는 편성 점수에서 제외
- 팟마다 평균 전투력이 표시됨
- 인원이 모자라면 "미완성" 팟으로 표시하고 구인 슬롯을 보여줌
- 결과는 텍스트로 복사해서 카톡/디스코드에 공유 가능
